TEACH (APIs and Syntax) ------------------------------------ // ===== LOCALSTORAGE ===== ~5mb // - Stores data with no expiration (persists even after browser is closed). // - Data is stored as STRING key-value pairs (must stringify objects). // - Size limit ~5MB (varies by browser). // Common Methods: localStorage.setItem("theme", "dark"); // Save data. // if name already exists then update let theme = localStorage.getItem("theme"); // Retrieve data localStorage.removeItem("theme"); // Remove a key localStorage.clear(); // Clears all keys // Storing Objects (convert to JSON string) let user = { name: "Rahul", age: 25 }; localStorage.setItem("user", JSON.stringify(user)); let parsedUser = JSON.parse(localStorage.getItem("user")); // Retrieve & parse //------------------------------------------------------------ // ===== SESSIONSTORAGE ===== ~5mb // - Works the same as localStorage but persists ONLY for the current tab/session. temporarily // - Data is cleared once the tab is closed. // - Use for temporary data like form progress. sessionStorage.setItem("sessionId", "abc123"); console.log(sessionStorage.getItem("sessionId")); sessionStorage.removeItem("sessionId"); //------------------------------------------------------------ // ===== COOKIES ===== // - Small text files (usually <4KB). // - Sent to server with every HTTP request (used for authentication). // - Can be set with expiration, domain, and path. // - Manual handling of expiry/encoding is needed. document.cookie = "username=Rahul; path=/; max-age=3600"; // key=value; path specifies scope; max-age=3600 (1 hour) // Reading all cookies (returns a string) console.log(document.cookie); // Deleting a cookie (set expiration in the past) document.cookie = "username=; expires=Thu, 01 Jan 1970 00:00:00 UTC; path=/;"; //------------------------------------------------------------ ------------------------------------ 2. MINDSET (When to Use What) ------------------------------------ // - Use localStorage for persisting UI preferences (dark mode, language, saved cart). // - Use sessionStorage for data specific to a tab (multi-step form progress). // - Use cookies for server communication (auth tokens, sessions across tabs). ------------------------------------ 4.
